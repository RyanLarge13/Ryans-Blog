Dynamically Typed Language

Javascript is a dynamically typed language. This is extremely important to understand so let me make it clear with examples of what I mean by a dynamically typed language and how it works. There are two different ways a language can determine its types of variables:

  • Dynamically
  • Statically

Statically Typed

Here is an example of a statically typed variable in a language like C++, C, Java, Go, Rust, etc.. etc.... etc....... the cool languages you know? And don't sit here judging, I know I left out a lot of languages that are cool and statically typed if you do know a little about what I mean, and of course if I spent the time writing them out instead of writing this last sentence defending my position for not writing these other languages out I could have avoided writing this long nonsense last sentence, but nah...

int myNumber = 24;

What are you looking at here? Lets break it apart. We are going start by reading this code snippet backwards. I am using C in this example, and many languages such as JavaScript, C++ and more use the same conventions as in C where a ";" is used at the end of a statement for what they call a statement delimiter, helping a compiler determine the end of one statement of code to the next.


- Now, just to the left of our semicolon we see a numeric value. Here we put the value 24.


- To the left of this we have our assignment operator "=", this will take out numeric value and place it within a variable


- And to the left of the assignment operator we give our value a name. We are going to name the value of 24 as the text "myNumber"


- Finally we have the part of this statement that actually matters. "int"..


So, what is int? Simple, int is out static type definition. We are telling the compiler that our variable myNumber is of a specific type. An integer type.


Incase you did not already realize int is short for integer that is the case a matter of fact. There are a few reasons we want to specify the type of a variable, but we will go over more of that later. For now you understand that before initializing a variable we need to define its type in a statically typed language like C.

So what is a const type and if const is a type and we intentionally type out this type how is it any different at all from the statically typed example? Great question. For the sake of your sanity the const declaration is not a type. It is only a way to say "hey, this variable myNumber is not allowed to be reassigned to any new value, EVER.


This means that in JavaScript, our myNumber variable is not assigned to be of a certain type.
